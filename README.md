# 📚 Simple Library Database

A relational database design for a Simple Library Management System that manages books, book copies, users, borrowing and return operations, reservations, and fine management.

The project was designed using **ERD (Entity Relationship Diagram)** and **Relational Schema** principles, with a focus on efficient data organization, relationship modeling, and normalization to support real-world library operations.

## 🚀 Features

* Book Management
* Multiple Book Copies Tracking
* User Management
* Borrowing & Return Management
* Holds & Reservations System
* Fine & Payment Tracking

## 🧠 Concepts Practiced

* Database Design
* ERD Modeling
* Relational Schema Design
* Primary & Foreign Keys
* One-to-One, One-to-Many, and Many-to-Many Relationships
* Database Normalization
* Business Requirements Analysis

## 🛠️ Tools & Technologies

* SQL Database Design
* ERD Modeling
* Relational Database Concepts

This project was built as part of my database design and SQL learning journey, focusing on converting real-world library requirements into a structured, scalable, and maintainable relational database design.
